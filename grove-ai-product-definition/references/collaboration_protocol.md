# 多智能体协作协议

## 1. 单一共享状态

所有智能体仅通过运行目录交换结构化结果。聊天中的临时结论不视为正式状态。

## 2. ID 体系

- `E-*`：Evidence
- `C-*`：Claim
- `P-*`：Product
- `J-*`：Job
- `O-*`：Opportunity
- `K-*`：Concept
- `R-*`：Risk
- `X-*`：Experiment
- `H-*`：Handoff
- `D-*`：Decision

## 3. 文件所有权

- 原始证据只追加，不覆盖；
- 编码修订以版本号保存；
- 任何删除必须写入变更日志；
- 红队不得修改提案者原稿，只提交反驳文件；
- Evaluation Auditor 只读原始证据和预测。

## 4. 交接最小字段

- 目标；
- 输入 ID；
- 预期输出；
- 已知假设；
- 不确定性；
- 阻塞项；
- 接收人；
- 截止 Gate；
- 完成定义。

## 5. 冲突裁决

依次检查：

1. 是否同一市场和同一产品版本；
2. 是否同一时间窗口；
3. 是否官方规格与实际测量口径不同；
4. 是否样本偏差；
5. 是否存在固件更新；
6. 是否为个案；
7. 是否能通过实验复现。

无法裁决时保留冲突，不强制合并为单一结论。

## 6. 人工决策点

- Gate G：证据范围与可信度；
- Gate R：核心机会；
- Gate V：回测是否成立；
- Gate E：概念与规格；
- Launch Gate：是否进入样机或商业化。

## 7. 推荐并行方式

第一轮并行：
- 官方规格与标准；
- 独立评测；
- 评论与社区；
- 市场和价格；
- 安全与合规。

第二轮并行：
- JTBD；
- 工程约束；
- 商业约束；
- 合成用户边界测试。

第三轮串行：
- 概念收敛；
- 红队；
- 回测审计；
- 人工 Gate。
