# 最小运行示例

## 任务

对 10,000mAh 磁吸无线充电宝进行历史桌面回测，以 Anker 为核心品牌，控制容量、磁吸形态和目标设备。

## 最小数据

- 3 款 Anker 同品类产品；
- 1 款外部品牌控制；
- 100 条以上去重用户反馈；
- 官方规格、独立评测、零售反馈和社区讨论；
- 至少一个明确历史截止日期。

## 预测输出

```yaml
case_id: BT-001
core_jtbd: >
  当用户在通勤或短途出行中无法确定手机能否撑到目的地时，
  希望快速获得可信的剩余补电范围，同时避免线材和过热带来的不确定性。
top_pains:
  - PB-TRUST
  - PB-HEAT
  - PB-BULK
predicted_tradeoffs:
  - 更强屏显会增加结构空间和成本
  - 更高无线功率会扩大热设计压力
statement_type: HYPOTHESIS
```

## 概念约束

- 一个核心 JTBD；
- 最多三个差异化；
- 规格均为假设；
- 回测预测阶段不得查看待测产品发布后的资料；
- 解封后用后续评测和反馈检验，而不是以“产品是否上市”为唯一成功标准。

## 输出检查

- 证据可追溯率 100%；
- 泄漏率 0；
- 有外部品牌控制；
- 有反证；
- 有消融；
- 有失败案例；
- 有工程验证计划。
