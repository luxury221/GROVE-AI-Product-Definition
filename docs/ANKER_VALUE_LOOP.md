# 安克业务价值闭环

```mermaid
flowchart LR
    A[全球用户与市场证据] --> B[GROVE-AI 统一编码]
    B --> C[产品机会与工程约束]
    C --> D[人工产品决策 Gate]
    D --> E[新品定义与 MVP]
    E --> F[结果与知识资产沉淀]
    F --> B
```

## 闭环责任

| 环节 | 关键问题 | 责任产物 |
|---|---|---|
| 证据进入 | 来源、版本、时间和市场是否清楚？ | Evidence Ledger |
| 统一编码 | FACT、INFERENCE、HYPOTHESIS 是否分离？ | 编码表、冲突记录 |
| 机会与约束 | 机会是否有支持与反证？工程边界是否前置？ | Opportunity Score、Risk Register |
| 人工 Gate | 谁批准、拒绝或要求补证？ | Decision Record |
| MVP | 最小实验能否证伪核心假设？ | Experiment Plan |
| 资产沉淀 | 哪些证据、规则和失败案例可复用？ | 品类适配器、案例库、版本记录 |

飞书仅作为 `Planned` 的协作载体承载这些记录；GROVE-AI 的核心是证据纪律、决策机制和验证协议。
