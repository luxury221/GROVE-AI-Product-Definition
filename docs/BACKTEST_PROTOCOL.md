# GROVE-AI 正式历史回测协议

> 状态：`Completed`（协议已定义）；正式数据收集与回测执行为 `Planned`。

## 1. 预注册边界

每个回测案例必须在预测前冻结：

- `case_id`、品类、市场和价格带；
- 历史截止日期与观察窗口；
- 同品类控制变量；
- 来源白名单和排除规则；
- 外部品牌控制；
- 留出产品或产品族；
- 基线、指标、消融和通过阈值；
- 负责人工 Gate 的决策人。

预测快照生成后保存时间戳和内容哈希。任何修改都创建新版本，不覆盖原预测。

## 2. 数据隔离

```text
预测区：只含 historical_cutoff 当日及以前可获得的信息
观察区：只在预测冻结后解封，用于后验检验
审计区：保存来源时间、访问时间、版本和泄漏记录
```

如无法证明资料在截止日前可获得，则不得进入预测区。泄漏率目标为 0；发现泄漏时，该案例必须标记失败或重新预注册，不能静默删除记录。

## 3. 对照设计

至少包括：

1. 简单评论频次或单模型总结基线；
2. 一个外部品牌控制；
3. 一个留出产品或产品族；
4. 一个移除关键模块的消融；
5. 一个失败或被否决案例。

## 4. 指标

| 指标 | 说明 |
|---|---|
| Pain Recall@K | 预测 Top-K 痛点在后续观察中的召回 |
| Opportunity Precision@K | Top-K 机会中获得后续多源支持的比例 |
| Trade-off Detection Rate | 预测取舍后来被独立观察到的比例 |
| Contradiction Resolution Rate | 冲突被记录并形成决策的比例 |
| Evidence Traceability | 关键结论可回溯到合格 Evidence ID 的比例 |
| Leakage Rate | 使用截止日后信息的记录比例 |
| Ablation Delta | 完整流程相对删减模块的指标变化 |

指标阈值必须在解封观察数据前写入 `run_manifest.yaml`。

## 5. 失败报告

所有失败必须保留，并区分：数据不足、时间泄漏、编码偏差、公式偏置、角色缺失、工程约束遗漏或机会不可区分。修改 Skill 后需说明是否可能对历史案例过拟合。

## 6. 完成定义

只有在以下条件同时满足时，正式回测才能标为 `Completed`：

- 预测与观察数据隔离可审计；
- 泄漏率为 0；
- 对照、留出、失败案例和消融齐全；
- 指标可由仓库脚本复算；
- 结果包含负面发现；
- Evaluation Auditor 与人工 Gate 已签署决策记录。
