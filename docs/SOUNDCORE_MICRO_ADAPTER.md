# Soundcore 真无线耳机微型适配器

> 目标：以最小案例展示 GROVE-AI 的跨品类迁移设计。
>
> - **Status:** `In Progress`
> - **Design artifact:** `Completed`
> - **Empirical validation:** `Planned`
> - 产品机会与规格：`Hypothesis`

本适配器属于入围后的挑战目标，不作为充电宝主回测闭环完成的前置条件。

## 1. 品类边界

研究对象限定为面向通勤、会议与运动场景的 Soundcore 真无线降噪耳机。开放式耳机、头戴式耳机、助听设备和专业监听耳机不直接纳入主基准集，除非作为外部控制。

## 2. 三个场景化 JTBD

1. **通勤**：当环境噪声和连接条件不断变化时，希望持续听清内容且不需要频繁调节模式；
2. **会议**：当在共享空间参加长时间通话时，希望自己的声音稳定清晰，并能可信判断剩余会议续航；
3. **运动**：当出汗、震动和风噪同时出现时，希望耳机保持佩戴稳定、连接稳定与安全环境感知。

正式产品定义阶段必须通过证据 Gate 从三者中只保留一个核心 JTBD。

## 3. 十个关键规格字段

| 字段 | 记录要求 |
|---|---|
| 声学单元与调音目标 | 区分硬件规格与主观调音主张 |
| ANC 频段与场景表现 | 不只记录单一降噪深度数字 |
| 通透与风噪控制 | 记录模式、延迟和户外表现 |
| 单次/总续航 | 记录 ANC、编码、音量和测试口径 |
| 快充能力 | 充电时间与可播放时间口径 |
| 左右耳电量偏差 | 记录使用模式和循环阶段 |
| 编解码与设备兼容 | 区分地区、系统和固件版本 |
| 连接稳定性与多点 | 记录干扰环境、切换延迟和失败模式 |
| 佩戴稳定与防护等级 | 耳型、耳帽、运动强度与 IP 口径 |
| 通话与听力安全 | 麦克风场景、声压提醒与区域规则 |

## 4. 八个痛点代码

| 代码 | 痛点 |
|---|---|
| `TWS-FIT` | 长时佩戴压痛、松动或耳型不适配 |
| `TWS-ANC` | 降噪在风噪、人声或突发声下不稳定 |
| `TWS-CALL` | 复杂环境通话拾音差 |
| `TWS-LINK` | 断连、多点切换或音画同步问题 |
| `TWS-BATT` | 标称续航与真实场景差异大 |
| `TWS-IMBALANCE` | 左右耳电量或声学表现不一致 |
| `TWS-CONTROL` | 触控误触、模式层级复杂 |
| `TWS-SAFETY` | 音量暴露、环境感知或睡眠佩戴风险 |

## 5. 候选机会：ContextPilot（Hypothesis）

用设备端可解释状态模型统一“场景续航预测 + 模式切换建议”，根据 ANC、编码、音量、麦克风、连接数量和左右耳衰减，给出剩余可用场景范围，例如“可完成约 2 场 45 分钟会议”，并在置信度不足时回退到保守估计。

这不是自动堆叠更多模式。必须验证它是否比电量百分比和固定模式更容易理解、是否减少会议中断，以及模型是否优于简单阈值规则。

## 6. 跨品类迁移对照

| 充电宝适配器 | 真无线耳机适配器 | 可复用的 GROVE-AI 机制 |
|---|---|---|
| 功率、温升、容量 | 音质、降噪、续航 | 多目标约束与口径统一 |
| 磁吸对位 | 佩戴稳定性 | 物理适配的场景化证据 |
| 有线/无线效率 | 编解码与连接稳定性 | 模式与环境交互 |
| 电芯老化 | 电池与左右耳衰减 | 生命周期与自校准 |
| 航空与 3C | 听力安全与区域认证 | 标准、地区与版本追踪 |
| RangeView | 场景续航预测 | 从百分比转为可理解结果 |
| RouteCharge | 通勤/会议/运动策略 | 设备端策略选择与解释 |

## 7. 最小验证包

- 为三个场景分别收集官方规格、独立评测和真实反馈；
- 选择一个核心 JTBD，其他场景作为边界条件；
- 建立至少一个外部品牌控制与一个留出产品族；
- 冻结历史截止日后再生成机会；
- 用后续固件、产品和反馈检验连接、续航与佩戴风险；
- 记录被否决的“模式堆叠”反例和否决证据。
